I recall struggling with the SQL queries when I took the DBMS course. As the labs progressed and the queries got more complicated, I started doubting the correctness of my queries. Knowing either way would have helped me realize my misconceptions. After taking the db-class course, and seeing their use of RA workbenches, I had cues on how to solve this problem. I talked about this to Professor PM Jat, and he was very supportive of the idea.

While we realize this is pretty basic, we hope that you find it useful. You can write down the issues with the system at my github repository. InteractiveSQL is open-source, but not quite ready for others to jump in the development fray. This would change on the next bump to the version number, which I hope will happen soon.

Stack: We are using psql 8.3.11 (PostgreSQL) for executing SQL queries. The Relational Algebra queries are translated into PostgreSQL using RA interpreter (developed by Prof. Jun Yang at Duke University). All the nice styling is due to Twitter Bootstrap. The horrible color choices are mine.
